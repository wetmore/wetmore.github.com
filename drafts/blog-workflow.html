My workflow for this blog

This post is currently a draft, and does not show up in the archive

  1. Pandoc’s Markdown parser is one of the most powerful out there, and supports many different extensions.

This website is built with Hakyll, a static site generator written in Haskell. Hakyll is powered by Pandoc, which can convert content between a variety of different formats. In particular, Pandoc can convert Markdown1 with LaTeX math into HTML. This means I can write my blog posts in Markdown with math included without much of a hassle.

  1. Domain-Specific Language

The file site.hs is the main specification of how the site is built. Hakyll provides a nice DSL2 for describing how to process the posts and other content into a bunch of HTML and CSS files the site consists of. We can compile site.hs with ghc --make site.hs and use it for a number of different commands:

  • /site build: Build the static site into the folder _site.
  • /site clean: Clear the site cache and delete _site.
  • /site rebuild: Runs clean, then build.
  • /site preview: Serve the site on localhost:8000 and rebuild when files change.
